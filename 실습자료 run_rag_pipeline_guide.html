🎯 팀 실습 실행 순서 — 4개 명령어 + 2개 브라우저

각 팀이 이 순서대로 진행하세요. 처리·인덱싱은 dry-run으로 미리보기, 검색은 실제 실행, 마지막에 브라우저 2개로 시각적 확인합니다.

2

3
인덱싱(미리보기) — python 02_indexer.py . --dry-run

# 팀 실습 — 순서대로 실행 (CLI 4단계) $ python 01_html_processor_v4.py . --dry-run $ python 03_md_processor.py . --dry-run $ python 02_indexer.py . --dry-run $ python 03_search.py "모터제어" # 브라우저 확인 2개 # http://192.168.0.75:5601/ ← OpenSearch Dashboards # http://192.168.0.75:8501/ ← 피지컬 AI 지식 시스템 (Streamlit)

왜 dry-run만? 팀별로 여러 명이 동시에 실행하면 같은 인덱스에 중복 쓰기가 발생할 수 있어요. 미리보기(dry-run)로 각 단계에서 어떤 데이터가 어떻게 처리되는지 관찰·이해하는 것이 오늘 실습 목표입니다. 실제 인덱싱은 강사가 통합 실행합니다.
🔎
검색·브라우저는 자유롭게 반복 Step 4의 python 03_search.py, Step 5·6의 브라우저 확인은 모두 읽기 요청이라 팀 간 간섭이 없습니다. 여러 검색어로 반복해보고, Dashboards와 Streamlit UI에서 결과가 어떻게 다른지 비교해보세요.
🎓
CLI → GUI 학습 흐름의 의미 CLI(03_search.py)는 내부 동작을 보여주고, Dashboards(5601)는 저장 구조를 보여주고, Streamlit(8501)은 최종 사용자 경험을 보여줍니다. 같은 데이터를 세 가지 관점에서 관찰하면 RAG 시스템의 전체 그림이 잡혀요.

# 임베딩관 PC에 SSH로 접속 후 $ sudo systemctl status ollama $ sudo systemctl restart ollama
