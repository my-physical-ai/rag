🕷️ 크롤러 (Crawler) — 자료를 모아오는 로봇

인터넷의 좋은 자료를 자동으로 찾아서 가져와요

🐝
비유: 꿀벌의 꿀 수집
꿀벌이 여러 꽃을 돌아다니며 꿀을 모아오듯, 크롤러는 여러 웹사이트를 돌아다니며 새로운 글, 뉴스, 기술 문서를 모아와요. 매일 새벽 3시에 자동으로 출발합니다!
💡 실제로는 — 08_crawler.py가 RSS 피드(웹사이트의 새 글 알림)를 읽어서 새로운 글이 있으면 다운로드합니다.

🪞 미러링 (Mirroring) — 자료를 정리하는 비서

수집한 자료를 AI가 이해할 수 있는 형태로 변환해요

📋
비유: 도서관 사서의 책 분류
새 책이 도서관에 도착하면 사서가 제목 붙이기, 분류번호 매기기, 요약 쓰기, 선반에 꽂기를 해요. 미러링도 마찬가지! 크롤러가 가져온 날것의 자료를 깔끔하게 정리해서 OpenSearch 서랍에 넣을 준비를 합니다.
💡 실제로는 — 12개 파일로 구성된 외부 지식 미러링 아키텍처가 동작합니다. HTML 파싱, 텍스트 청킹, 메타데이터 추출, 벡터 변환을 수행해요.

미러링이 하는 일 — 3단계

✂️ 청킹 (Chunking) — 자료를 적당한 크기로 잘라요

긴 문서를 한 번에 읽으면 AI가 헷갈려요. 단락이나 제목 단위로 적당한 조각으로 나눕니다. 한 조각이 한 개의 "검색 카드"가 돼요.

🏷️ 태깅 (Tagging) — 라벨을 붙여요

각 조각에 "이건 LeKiwi 관련", "이건 캘리브레이션 관련" 같은 태그를 붙여요. 나중에 카테고리별로 찾기 쉽게요!

🔢 벡터화 (Vectorization) — 숫자로 바꿔요

AI는 글자를 못 읽어요. 대신 1024개의 숫자 배열로 바꾸면 "이 문장과 저 문장이 비슷한지" 계산할 수 있어요. 이걸 임베딩관(BGE-M3)이 해요!

🎵
벡터화를 더 쉽게!
음악 앱에서 "신나는 노래 틀어줘"라고 하면, 앱이 노래 제목을 검색하는 게 아니라 노래의 분위기(빠른 템포, 밝은 음색)를 숫자로 비교해서 비슷한 노래를 찾아줘요. 벡터화도 마찬가지! 글자 대신 "의미의 숫자"로 비교하는 거예요.
3

📚 OpenSearch — AI의 기억 서랍장

정리된 자료를 보관하고, 질문이 오면 빠르게 찾아줘요

🗄️
비유: 마법의 서랍장
보통 서랍장은 라벨로만 찾죠 ("수학 노트"를 찾으려면 "수학" 서랍을 열어야 해요). 하지만 OpenSearch는 두 가지 방법을 동시에 써요: ① 라벨로 찾기 (BM25) + ② "이것과 비슷한 내용 찾기" (kNN). 두 방법을 합쳐서 가장 좋은 결과를 골라요!
💡 실제로는 — BM25는 Nori 한국어 형태소 분석기로 키워드를 매칭하고, kNN은 BGE-M3 벡터의 코사인 유사도를 계산합니다.

두 가지 검색 방법 비교

🔤BM25 (키워드 검색)
방식단어가 일치하는지 확인
장점"STS3215" 같은 정확한 코드명
약점"로봇 팔" ≠ "매니퓰레이터"
비유도서관에서 제목으로 찾기
비중40%
🧠kNN (의미 검색)
방식뜻이 비슷한지 계산
장점"로봇 팔" ≈ "매니퓰레이터"
약점정확한 코드명은 놓칠 수 있음
비유사서에게 "이런 느낌 책" 요청
비중60%
⚖️
비유: 국어 + 수학 시험 합산
국어 100점, 수학 500점 만점이면 그냥 더하면 수학이 압도해요. 둘 다 0~1점으로 환산한 뒤 합쳐야 공정해요. 이걸 해주는 게 hybrid_pipeline이에요! BM25 점수와 kNN 점수를 0~1로 바꾼 뒤, BM25×0.4 + kNN×0.6 으로 합산합니다.
🔍
HyDE 기법도 함께! — "캘리브레이션"이라는 짧은 질문으로는 좋은 자료를 못 찾을 수 있어요. 그래서 준비관이 먼저 "이런 답변이 있을 것 같아"라는 가상 문서를 만들고, 원본 질문 + 가상 문서 두 가지로 검색합니다. 그러면 더 다양한 자료가 나와요!
4

🧠 4-Hall — AI의 4개의 뇌

각각 다른 역할을 맡은 4개의 GPU 서버가 협력해요

🏥
비유: 병원의 전문의 4명
아플 때 한 명의 의사가 모든 걸 다 하면 힘들죠? 병원에는 역할이 나뉘어 있어요:
접수처(준비관)가 증상을 정리하고 → 검사실(임베딩관)이 검사를 하고 → 주치의(생성관)가 진단서를 쓰고 → 감독의(검수관)가 진단이 맞는지 확인해요.
📋 1단계 — 준비관 (RTX 4090 · Qwen3:14B)
질문 분석
HyDE 가상 문서 생성
검색 키워드 확장
▼
🔍 2단계 — 임베딩관 (RTX 3070 · BGE-M3)
텍스트→벡터 변환
BM25+kNN 하이브리드 검색
상위 5개 자료 선택
▼
✍️ 3단계 — 생성관 (RTX 5090 · Qwen3:32B)
검색 자료 읽기
자료 기반 답변 작성
출처 번호 표기
▼
🔬 4단계 — 검수관 (RTX 4070 · Gemma3:12B)

신뢰도 점수 (0~100)
🚨
왜 생성관과 검수관은 다른 모델을 쓸까?
시험에서 내가 쓴 답을 내가 채점하면 어떨까요? "아, 이거 맞는 것 같은데?" 하고 넘어가기 쉬워요. 그래서 Qwen이 답을 쓰면 Gemma가 채점합니다. 서로 다른 가문(패밀리)의 AI라서 편향 없이 냉정하게 판단할 수 있어요. 이걸 교차 패밀리 검증이라고 해요!

🎓 오늘 배운 핵심 5가지

1. 크롤러는 매일 새벽 인터넷에서 새 자료를 자동으로 모아와요
2. 미러링은 날것의 자료를 청킹(자르기) → 태깅(분류) → 벡터화(숫자 변환)해요
3. OpenSearch는 키워드(BM25) + 의미(kNN) 두 가지로 동시에 검색해요
4. 4개 GPU 서버가 각각 준비 → 검색 → 생성 → 검증 역할을 나눠요
5. Qwen이 답을 쓰고 Gemma가 채점하는 교차 패밀리 검증으로 환각을 잡아요
