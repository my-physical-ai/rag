그 다음 "🚢 조선 트렌드" 메뉴 클릭:

→ ⚠️ 아직 외부 소스를 추가하지 않았으므로 결과가 부족할 수 있음

6
🌐 전체 통합 — 모든 팀 자료가 합쳐지는 순간
사이드바에서 "전체 통합"을 선택하면 4팀이 추가한 모든 자료가 통합 검색됩니다.

사이드바 → 🌐 전체 통합 → 외부 트렌드

질문: "로봇과 AI로 산업이 어떻게 변하고 있나요?"

이 질문에 대해 4팀이 추가한 모든 자료에서 답변이 합성됩니다:
• 조선 소스에서: 스마트 조선소 사례
• 제조 소스에서: 스마트팩토리 예지보전
• 물류 소스에서: 물류 로봇 AMR 도입
• 본사 소스에서: 전사 AI 전환 전략
🧠
패러다임 전환 — "AI 검색"이란?
기존 검색: 키워드가 일치하는 문서만 반환
AI 검색: 의미를 이해하고, 여러 출처를 합성하여, 검증까지 거친 답변을 생성

각 팀이 다른 업종의 소스를 추가했지만, 전체 통합에서는 한 질문에 대해 조선+제조+물류+전략이 모두 반영된 답변이 나옵니다. 이것이 이중 RAG의 힘입니다.
🔧 자주 발생하는 문제
증상원인해결
크롤러 에러: KeyError: 'query'RSS 항목이 google_news 섹션에 잘못 들어감RSS는 rss_feeds:에, 뉴스는 google_news_topics:에
크롤러 에러: RSS 파싱 실패해당 사이트 XML 오류다른 RSS URL로 교체
외부 트렌드 메뉴에서 결과 없음인덱싱을 안 했거나 인덱스가 다름10_bulk_ingest.py 실행 확인
답변이 중간에 잘림토큰 제한 (2048) 부족max_tok=4096으로 변경
4팀 동시 질의 시 느림Ollama 순차 처리팀별 시간차 두기 또는 라운드제
원본 보기가 파일 목록 표시url 필드를 못 가져옴search_engine.py의 _source에 url 추가
✅ 오늘 배운 것
1. 09_sources.yaml에 URL을 추가하면 → 크롤러가 자동으로 수집한다
2. 수집된 자료는 10_bulk_ingest.py로 → OpenSearch에 인덱싱된다
3. 인덱싱된 자료는 즉시 → 4-Hall RAG 파이프라인에서 검색 가능하다
4. 팀별 메뉴에서 업종별 트렌드를 → 각자 다른 관점으로 검색할 수 있다
5. 전체 통합에서는 → 모든 팀의 지식이 합쳐진 답변이 나온다
6. AI가 답변을 만들면 → 다른 AI가 교차 검증해서 환각을 잡는다
7. 이것이 → 폐쇄망에서도 동작하는 온프레미스 AI 검색 시스템이다
